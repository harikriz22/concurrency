Concurrency
vs Parallelism

Two ideas that get confused constantly — one is about handling many things at once, the other is about doing many things at once. Same words, different physics.

→ Use the arrow keys, click either edge, or tap a dot to move through
01 — Definition

Concurrency

Managing multiple tasks during the same period of time.

→

The tasks appear to run together because the CPU switches between them quickly.

→

Nothing is happening at the exact same instant — progress is interleaved, not simultaneous.

→

One worker, many things in flight. It's about structure, not speed.

02 — Definition

Parallelism

Running multiple tasks at exactly the same time.

→

This requires multiple CPU cores — or multiple processors — actually working simultaneously.

→

Every worker is busy at the same instant. It's about raw throughput.

→

Parallelism is one way to achieve concurrency — but concurrency doesn't require it.

05 — Recap

Concurrency: The Habit of Switching

Concurrency means multiple tasks make progress during the same time period — by switching, not by simultaneity.

01

The chef starts cooking Order 1. While it's boiling, instead of standing and watching the pot, he starts Order 2.

02

While Order 2 bakes, he cuts vegetables for Order 3 — constantly switching between tasks.

03

Even something as ordinary as making tea works this way: kettle on, then you don't stand there — you get the cup and teabag ready while it heats.

06 — Motivation

Why We Want Concurrency

Programming has the same problem as the kitchen. Some tasks spend most of their time waiting; others spend all their time calculating.

I/O-bound — mostly waiting

  • Downloading files
  • Reading from a database
  • Network requests
  • Waiting on user input

CPU-bound — mostly calculating

  • Image processing
  • AI / model inference
  • Encryption
  • Sorting huge datasets
07 — Python's Toolbox

Different Tools for Different Waits

Python gives different tools for different situations — pick based on whether a task is waiting or calculating.

01

Threading

Multiple threads inside one process, switching quickly. Great for I/O-bound waiting.

02

Multiprocessing

Multiple independent processes, each on its own core. True parallelism for CPU-bound work.

03

Asyncio

A single thread that cooperatively switches between waiting tasks — concurrency without extra workers.
