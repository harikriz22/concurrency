Concurrency vs Parallelism
___________

Concurrency

Managing multiple tasks during the same period of time.

The tasks appear to run together because the CPU switches between them quickly.

Parallelism

Running multiple tasks at exactly the same time.

This requires multiple CPU cores (or processors).

eg: concurrency 

Imagine a restaurant.

There is only one chef.

Three customers order food.

Pasta
Pizza
Burger


The chef starts cooking the pasta.

While the pasta is boiling, the chef starts preparing the burger.

While the burger is grilling, the chef prepares the pizza dough.

The chef keeps switching between tasks.


Only one chef is working.

Nothing is truly happening simultaneously.

But all three dishes make progress.

This is Concurrency.

Parallelism (Three Chefs)

Now imagine there are three chefs.

Chef 1 → Pasta

Chef 2 → Pizza

Chef 3 → Burger

All three dishes are cooked at the same time.

This is Parallelism.







Concurrency
________

Concurrency means multiple tasks make progress during the same time period.


The chef starts cooking Order 1.

Instead of standing and watching the pot...

He keeps switching between tasks.
